AI Frontend Engineer — LLM Interfaces & Generative UI

I build the interface layer for AI: streaming chat UIs, generative interfaces and agent-driven UX that make LLMs feel fast, clear and trustworthy — with React, TypeScript and the Vercel AI SDK.

AI products live or die on the interface. A great model with a confusing UI feels broken; a modest model with a thoughtful UI feels magical. I specialize in that layer — streaming, tool surfaces, loading and error states, and UX that sets honest expectations.

How I work

  • Understand. I start with the real problem, users and constraints — not a feature list. A short discovery avoids building the wrong thing.
  • Plan. A clear plan: scope, non-goals, the approach, and how we'll verify it works. You always know what's being built and why.
  • Build. Small, tested, verifiable increments. Working software early and often, with clean, maintainable code that follows the project's conventions.
  • Ship & verify. Real deployment, real testing, honest reporting. I verify behavior end-to-end before calling anything done — no 'it should work'.

Can you connect a UI to my model or API?
